# Streamlit Frontend Use Cases

## 0. 결정된 방향

- 이번 스코프의 프론트엔드는 Streamlit 기준으로 진행한다.
- 백엔드에서 전달받는 메시지와 응답 구조를 사용자가 보기 좋은 화면으로 어떻게 보여줄지 먼저 정의한다.
- A2UI/AGUI 같은 후속 인터페이스 논의는 4차 시기 또는 이후로 미룬다.
- 사용자가 질문했을 때 화면에 어떤 정보가 보여야 하는지 먼저 정의한다.

### 우선 산출물

- 사용자 use case와 렌더링해야 할 화면 요소를 매핑한 표
- 질문 입력, 처리 상태, 답변, 근거/출처, 오류 메시지 표시 기준

## 1. 목적

Streamlit 기반 프론트엔드에서 사용자가 전체 서비스 흐름을 실행하고, 백엔드 API 및 RAG 결과를 확인할 수 있도록 필요한 화면과 상호작용 유스케이스를 정리한다.

## 2. 주요 액터

- 최종 사용자
- 프론트엔드 담당자
- 백엔드 API
- RAG 응답 모듈

## 3. 핵심 유스케이스

| ID | 유스케이스 | 사용자 행동 | 프론트엔드 책임 | 렌더링 화면 요소 |
| --- | --- | --- | --- | --- |
| UC-FE-01 | 사용자 입력 화면 제공 | 질문, 조건, 파일 또는 필요한 입력값을 제공한다. | 필수 입력값 누락 여부를 검증하고 피드백을 제공한다. | 질문 입력창, 조건 입력 영역, 파일 업로드, 입력 검증 메시지 |
| UC-FE-02 | 백엔드 API 호출 | 실행 버튼을 눌러 백엔드 처리 요청을 보낸다. | 요청 상태, 로딩 상태, 오류 상태를 화면에 표시한다. | 실행 버튼, 처리 상태 표시, 로딩 표시, 오류 알림 |
| UC-FE-03 | RAG 결과 표시 | RAG 검색/생성 결과를 확인한다. | 답변, 근거 문서, 참고 정보가 구분되도록 표시한다. | 답변 요약, 상세 답변, 근거/출처 패널, 참고 정보 탭 |
| UC-FE-04 | 결과 재실행 및 입력 수정 | 입력값을 수정하고 다시 실행한다. | 이전 결과와 새 결과가 혼동되지 않도록 상태를 갱신한다. | 재실행 버튼, 최근 실행 기준, 이전 결과 초기화 또는 접기 |
| UC-FE-05 | 모호한 질문 보정 | 짧거나 범위가 넓은 질문을 입력한다. | 버튼식 의도 선택 없이 assistant가 채팅으로 필요한 정보 범위를 되묻고 사용자의 후속 답변으로 이어지게 한다. | 채팅 메시지, 보충 질문, 후속 입력창 |
| UC-FE-06 | 통합 시나리오 검증 | 백엔드와 RAG가 연결된 전체 흐름을 실행한다. | 2026-06-04까지 연결 흐름을 실행 가능한 상태로 만든다. | 통합 테스트 화면, API 응답 상태, 원본 JSON 확인 영역 |

## 4. 화면 요소 표시 기준

| 화면 요소 | 표시 기준 | 권장 Streamlit 컴포넌트 |
| --- | --- | --- |
| 질문 입력 | 사용자가 질문을 작성하거나 조건을 바꾸는 기본 진입점 | `st.text_input`, `st.text_area`, `st.selectbox`, `st.file_uploader` |
| 처리 상태 | API 요청 시작부터 응답 완료 또는 실패까지의 상태 안내 | `st.status`, `st.spinner`, `st.info` |
| 답변 | 사용자가 가장 먼저 확인해야 하는 생성 결과 | `st.chat_message`, `st.info`, `st.markdown` |
| 상세 설명 | 긴 답변이나 단계형 설명을 접어서 제공 | `st.expander`, `st.tabs` |
| 표 데이터 | 수치, 비교, 조건 목록처럼 행/열 구조가 있는 결과 | `st.dataframe`, `st.table` |
| 근거/출처 | RAG가 참조한 문서, 법령, URL, 조문을 분리 표시 | `st.expander`, `st.tabs`, `st.markdown` |
| 오류 메시지 | 필수 입력 누락, API 실패, 응답 스키마 오류 발생 시 표시 | `st.error`, `st.warning` |
| 질문 보정 | 질문이 짧거나 핵심 조건이 부족할 때 표시 | `st.chat_message`, `st.checkbox`, `st.button`, `st.text_input` |
| 원본 응답 | 개발/검증 단계에서 백엔드 응답을 그대로 확인 | `st.json`, `st.code` |

## 5. 확인 필요 사항

- 화면별 필수 입력값
- 백엔드 API 엔드포인트와 응답 스키마
- RAG 결과에서 노출할 근거/출처 범위
- 에러 메시지 및 예외 처리 방식
