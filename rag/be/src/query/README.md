# Query Layer

`query/` is the Memgraph database-query layer for the RAG service.

It must contain only code needed to interact with Memgraph as a database. It is
not an agent runner layer, not an MCP server layer, and not a prompt directory.

## Boundaries

- `read/`: read/query methods backed by Memgraph capabilities and direct node
  lookup by unique id.
- `write/`: write/query methods for raw Cypher write execution and deterministic
  original-document registration.
- `utils.py`: small query value helpers shared by read/write methods.

There is no query service singleton. The singleton database connection belongs
to `external.memgraph.get_memgraph_bolt_client()`, and callers import the query
functions they need.

## Read Methods

Read methods are database query primitives. They can be wrapped both as internal
agent tools and as external MCP tools.

- `raw.py`: execute raw read queries.
- `document.py`: read any node by id and read `Document` nodes by document id.
- `schema.py`: read Memgraph schema with `SHOW SCHEMA INFO`.
- `text_search.py`: call Memgraph text search.
- `vector_search.py`: call Memgraph vector search.
- `traversal.py`: run bounded graph traversal helpers.

## Write Methods

Write methods are database query primitives for internal runtime use. They are
not exposed through external MCP.

- `cypher.py`: execute raw write Cypher generated by internal agents.
- `document_registration.py`: register the original uploaded document before
  agentic graph construction starts.

Chunk creation, relationship candidate creation, review note creation, and
actual edge materialization are not hard-coded here. Internal agents decide
those graph writes and call the raw write method through their write-capable
tools.

## Non-Goals

- Do not place agent prompts or MCP instructions in this directory.
- Do not add token-scanning guard files here.
- Do not add repository abstractions with ambiguous ownership names.
- Do not make service nodes duplicate agent-owned chunk or candidate writes.
- Do not add `dry_run`, mock execution branches, or job-progress tracking to
  query methods. Task status belongs to the ingest task layer.

## Tool Exposure Rule

Internal agent tools and pipeline services call functions from this layer.
External MCP receives only read methods. Write functions remain internal and are
reached through write-capable agent tools or deterministic
document-registration services.
