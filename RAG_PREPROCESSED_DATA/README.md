# RAG Preprocessed Data

이 디렉토리는 `RAG_ORIGINAL_DATA`에 남긴 RAG 대상 원본만 TOON 형태로 전처리한 결과입니다.

## 전처리 범위

- 기준 원본: `RAG_ORIGINAL_DATA/rag_datas`
- 전처리 결과: `RAG_PREPROCESSED_DATA/rag_datas`
- 법령: 고용/노동 관련 법령 5개 유지
- 조례: `조례/고령자_조례_raw.json` 중 고용/노동 관련 조례 4건만 조례별 TOON으로 분리
- 제외: 복지, 기초연금, 노인돌봄, 경로당, 노인여가, 노인일자리, 주거약자, 정보취약 계열 데이터
- 제외: 다운로드 manifest, 다운로드 스크립트, 임시 파일, cleaned markdown, 중복 전처리 JSON

## 토큰 산정 기준

- tokenizer: `cl100k_base`
- 산정 도구: `tiktoken`
- 비교 기준: 현재 정리된 `RAG_ORIGINAL_DATA` 원본과 `RAG_PREPROCESSED_DATA` TOON 결과
- 조례는 원본 1개 JSON에서 TOON 4개 파일로 분리되므로, 조례 그룹 aggregate 기준으로 비교했습니다.

## 해석

TOON 변환은 원본 JSON의 API 응답 메타데이터, 중복 래퍼, 비대상 필드, 복지 범위 데이터를 제거하고 RAG 검색에 필요한 문서 단위 필드만 남깁니다. 그 결과 전체 기준 약 13.1만 tokens, 40.81%가 줄었습니다.

조례는 법령보다 절감률이 낮습니다. 원본을 이미 고용/노동 관련 4건으로 좁힌 상태에서 조문 본문 비중이 높고, TOON에서는 조례별 분리와 출처 메타데이터를 유지했기 때문입니다.
